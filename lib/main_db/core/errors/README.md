# Ошибки в ядре БД (DBCoreError)

`main_db/core` использует независимую систему ошибок на базе пакета
`result_dart`. Это необходимо для того, чтобы модуль БД можно было использовать
изолированно, в том числе в CLI или фоновых изолятах, не привязываясь к слою
Flutter-приложения и `AppError`.

## Принципы

1. Все публичные методы сервисов возвращают `DbResult<T>` (alias для
   `ResultDart<T, DBCoreError>`).
2. В ядре не бросаются (`throw`) исключения в штатных ситуациях (вместо этого
   возвращается `Failure`).
3. Внутренние ошибки транзакций пробрасываются как `DBCoreError` напрямую и
   перехватываются на границе сервиса.
4. В `main_db/core` не импортируется `AppError`.

## Преобразование ошибок SQLite

Исключения из Drift/SQLite преобразуются в `DBCoreError` через `mapDbException`.
Названия SQL constraints маппятся в человекочитаемые ошибки через
`dbConstraintRegistry`.

Слой приложения может самостоятельно преобразовывать `DBCoreError` в `AppError`
перед показом UI.
