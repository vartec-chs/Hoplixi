# Как использовать ссылки на связанные объекты

## Быстрый старт

### 1. Создание ссылки

**В Quill редакторе:**

1. Установите курсор в нужное место (или выделите текст)
2. Нажмите кнопку **🔗** (ссылка) в панели инструментов
3. В модальном окне выберите объект из списка
4. Ссылка автоматически вставится в текст

### 2. Использование ссылки

**Для перехода к связанной записи:**

1. Кликните по ссылке в тексте заметки
2. Подтвердите открытие в диалоговом окне
3. Связанная запись откроется в режиме редактирования

## Примеры использования

### Пример 1: Создать ссылку на новом месте

```
1. Курсор в позиции: "Смотрите также: |"
2. Нажать кнопку ссылки
3. Выбрать объект "Проект Alpha"
4. Результат: "Смотрите также: Проект Alpha" (где текст - ссылка)
```

### Пример 2: Превратить текст в ссылку

```
1. Выделить текст: "[этой заметке]"
2. Нажать кнопку ссылки
3. Выбрать нужный объект
4. Результат: выделенный текст становится ссылкой
```

## Сценарии использования

### 📚 Создание базы знаний

Связывайте заметки с любыми связанными объектами: файлами, документами, паролями и другими заметками

### 📋 Управление проектами

Ссылайтесь на задачи, планы, документы и другие записи проекта

### 📝 Ведение журнала

Создавайте ссылки на предыдущие записи для контекста

### 🔖 Организация информации

Группируйте записи в сеть связанной информации

## Технические детали

**Основной формат ссылок:**

```
vault://entityType/itemId
```

Legacy формат `note://noteId` продолжает читаться.

**Визуально в тексте:**

```
Название объекта (кликабельная ссылка)
```

## Важно

- ✅ Ссылки сохраняются при экспорте заметки
- ✅ Можно создавать множество ссылок в одной заметке
- ✅ Note-to-note ссылки продолжают работать без изменений
- ⚠️ При удалении целевой записи ссылка останется, но не откроется

## Советы

разными типами записей 💡 Добавляйте контекст перед ссылкой для понимания связи 💡
Регулярно проверяйте актуальность связей
