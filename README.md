> **Hoplixi** - это современный, безопасный и удобный менеджер паролей для
> хранения, генерации и управления вашими учетными данными. Защитите свои
> онлайн-аккаунты с помощью мощного шифрования и интуитивного интерфейса.

## ✨ Особенности

### 🔒 Безопасность

- **Военное шифрование**: AES-256 и SQLite3 Multiple Ciphers для защиты баз
  данных
- **Шифрование файлов**: Все прикрепленные файлы и документы надежно шифруются
  алгоритмом XChaCha20 перед сохранением на накопитель
- **Биометрическая аутентификация**: Поддержка Touch ID / Face ID / Windows
  Hello
- **Локальное хранение**: Все данные хранятся исключительно локально на вашем
  устройстве
- **Мастер-пароль**: Дополнительный и главный уровень защиты вашего хранилища

### 🗂 Поддерживаемые сущности

Hoplixi — это не только менеджер паролей, а защищенное хранилище для разных
типов сущностей. Сейчас поддерживаются:

- 🔑 **Пароли**
- ⏱ **OTP / 2FA**: TOTP и (HOTP пока не реализован)
- 📝 **Заметки**
- 💳 **Банковские карты**
- 📄 **Документы**
- 📎 **Файлы**
- 👤 **Контакты**
- 🔐 **API-ключи**
- 🧰 **SSH-ключи**
- 📜 **Сертификаты**
- 🪙 **Криптокошельки**
- 📶 **Wi-Fi сети**
- 🪪 **Личностные данные**
- 🪪 **Лицензионные ключи**
- ♻️ **Коды восстановления**
- 🎁 **Карты лояльности**

Файлы и документы надежно защищены с помощью XChaCha20 шифрования, а их
метаданные (имя, размер, тип) сохраняются в зашифрованной базе данных.
Прикрепленные файлы доступны только после успешной аутентификации и расшифровки,
обеспечивая максимальную безопасность ваших данных.

Дополнительно поддерживаются типы пользовательских полей:

- Обычный текст
- Скрытое / чувствительное значение
- URL-адрес
- Адрес электронной почты
- Номер телефона
- Дата в формате ISO 8601
- Числовое значение

Категории и теги используют тот же набор сущностей, а также общий тип **mixed**
для смешанных записей.

### 🕰 Система истории (Версионирование)

Каждое изменение ваших данных бережно сохраняется благодаря встроенной системе
истории:

- **Полный аудит**: Отслеживание всех операций (создание, изменение, удаление)
  для каждой записи.
- **Снапшоты данных**: Сохранение предыдущих состояний полей (например, старых
  паролей или прошлых версий заметок).
- **Безопасность от ошибок**: Возможность в любой момент просмотреть историю
  изменений и восстановить случайно измененные данные.

### 🚀 Функциональность

- **Генерация паролей**: Создание сильных, уникальных паролей
- **Автозаполнение**: Быстрое заполнение форм в браузерах и приложениях (пока не
  реализовано)
- **Категоризация**: Организация записей по категориям и тегам
- **Поиск**: Мгновенный поиск по всем сохраненным данным
- **Импорт/Экспорт**: Поддержка различных форматов обмена данными
- **LocalSend**: Безопасная передача файлов и текста между устройствами по
  локальной сети без интернета

### 📡 LocalSend — Локальная передача данных

Hoplixi включает полностью готовый к работе модуль **LocalSend** для обмена
файлами и текстом между устройствами в одной сети — без интернета, облаков и
сторонних серверов.

#### Возможности

- **Автообнаружение устройств**: Устройства в локальной сети находятся
  автоматически через mDNS — никаких ручных настроек
- **P2P-передача через WebRTC**: Прямое соединение между устройствами без
  посредников
- **Файлы и текст**: Передача любых файлов и текстовых сообщений в рамках одной
  сессии
- **История обменов**: Все переданные и полученные файлы/тексты сохраняются в
  персистентной истории с возможностью удаления записей
- **Уведомление при разрыве**: При намеренном отключении одной стороны другая
  сторона мгновенно получает уведомление
- **Запрос подтверждения**: Принимающая сторона видит запрос на подключение и
  может принять или отклонить его

#### Как использовать

1. Откройте раздел **LocalSend** на обоих устройствах, находящихся в одной Wi-Fi
   сети
2. Нажмите на найденное устройство для подключения
3. Подтвердите запрос на принимающем устройстве
4. Отправляйте файлы или текст — они мгновенно доставляются напрямую
5. История всех обменов доступна через иконку 🕐 в правом верхнем углу

- **Android** и **iOS**: Мобильные приложения
- **Windows**, **Linux** и **macOS**: Десктопные версии

### 🔧 Дополнительные возможности

- **OTP Генератор**: Встроенный генератор одноразовых паролей (2FA)
- **QR-сканер**: Сканирование QR-кодов для быстрого добавления аккаунтов
- **Облачное синхронизирование**: Синхронизация между устройствами (в планах)
- **Виджеты**: Быстрый доступ к паролям с главного экрана
- **Темная тема**: Поддержка светлой и темной темы оформления (в будущем -
  кастомные темы)

### 🌐 Локализация

Приложение поддерживает два языка:

| Язык       | Код  | Статус                                                                |
| ---------- | ---- | --------------------------------------------------------------------- |
| 🇷🇺 Русский | `ru` | ✅ Основной язык, полностью переведён                                 |
| 🇬🇧 English | `en` | 🚧 В процессе — переведены все формы (формы ввода и просмотра данных) |

> Перевод остального интерфейса (навигация, настройки, главный экран и т.д.)
> продолжается.

Локализация реализована через [`slang`](https://pub.dev/packages/slang) —
типобезопасная кодогенерация переводов из JSON-файлов (`_ru.i18n.json`,
`_en.i18n.json`). Переводы доступны через `context.t` в виджетах или глобально
через геттер `t`. Язык сохраняется между сессиями и переключается без
перезапуска приложения.

## 📖 Использование

### Первый запуск

1. Установите мастер-пароль при первом запуске
2. Настройте биометрическую аутентификацию (рекомендуется)
3. Добавьте свои первые учетные данные

### Добавление нового пароля

1. Нажмите кнопку "Создать" на главном экране
2. Выберите категорию или создайте новую
3. Введите данные аккаунта
4. Используйте генератор паролей для создания сильного пароля
5. Сохраните изменения

### Генерация OTP

1. Перейдите в раздел "OTP Генератор"
2. Отсканируйте QR-код или введите секретный ключ вручную
3. Используйте сгенерированные коды для двухфакторной аутентификации

## 🔧 Разработка

### Структура одной единицы хранилища

```text
store_name/
│   store_key.json
│   store_manifest.json
│   store_name.hplxdb
├───attachments_decrypted/
└───attachments/
```

store_key.json - содержит salt и флаг useDeviceKey для генерации ключа
шифрования из пароля и/или устройства. store_manifest.json - содержит метаданные
хранилища (uuid, дату обновления). store_name.hplxdb - зашифрованная база данных
SQLite3 Multiple Ciphers. attachments_decrypted/ - временная папка для
расшифрованных файлов (может не существовать). attachments/ - папка для
зашифрованных файлов.

### Окружения (Flavors)

Проект использует пакет `flutter_flavorizr` для управления окружениями.
Настройки описаны в файле `flavorizr.yaml`. Доступно два окружения:

- **dev**: Для разработки (имя приложения "Hoplixi Dev", bundleId
  `com.hiplixi.app.dev`)
- **prod**: Для релиза (имя приложения "Hoplixi", bundleId `com.hiplixi.app`)

Для генерации настроек окружений используется команда:

```bash
flutter pub run flutter_flavorizr
```

### Сборка для релиза

Для удобной сборки релизных версий в проекте предусмотрен скрипт
`build_prod.bat`. Он позволяет выбрать платформу и автоматически выполняет
необходимые команды, включая инкремент номера сборки (через `cider`) для
Android.

Запуск скрипта сборки:

```bash
build_prod.bat
```

#### Генерация Keystore для Android

Для подписи Android APK необходим keystore. Проект включает скрипты для
генерации debug и production keystore.

**Генерация debug keystore:**

```bash
android\create_debug_keystore.bat
```

Этот скрипт создаст `android/app/debug.keystore` с предустановленными
параметрами для отладки.

**Генерация production keystore:**

```bash
android\create_keystore.bat
```

Этот скрипт создаст `upload-keystore.jks` в папке `android`. Вам будет
предложено ввести информацию для сертификата (имя, организация и т.д.).
**Обязательно сделайте резервную копию этого файла и храните его в безопасном
месте!**

#### Certificate fingerprints и package name для Android

Для настройки Android-приложения и OAuth-провайдеров нужны certificate
fingerprints и package name. Используйте следующие значения:

- Debug package name: `com.hiplixi.app.dev`
- Release package name: `com.hiplixi.app`

Чтобы получить fingerprints сертификата, перейдите в папку `android` и выполните
команду:

```bash
keytool -list -v -keystore my-release-key.jks -alias my-key-alias
```

Команда покажет поля:

- `SHA1`
- `SHA256`

После генерации keystore, убедитесь, что файл `android/key.properties` содержит
правильные пути и пароли:

```
storePassword=ваш_пароль
keyPassword=ваш_пароль
keyAlias=upload
storeFile=../upload-keystore.jks
```
