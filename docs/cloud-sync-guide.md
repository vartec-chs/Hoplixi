# Cloud Sync Guide

Подробная инструкция по snapshot-синхронизации в Hoplixi: что именно синхронизируется, как подключить облако, как понимать статусы и что делать при конфликте версий.

## Что такое Cloud Sync в Hoplixi

Cloud Sync в Hoplixi работает как snapshot-синхронизация текущего хранилища.

Это означает:

- в облако отправляется snapshot текущего store
- синхронизация не является live-редактированием одной и той же базы сразу с двух устройств
- приложение сравнивает локальную и удалённую версии и предлагает следующее действие

Практически это удобно для:

- переноса хранилища между устройствами
- резервной облачной копии текущего store
- восстановления или загрузки более новой удалённой версии

## Какие провайдеры поддерживаются

Сейчас в интерфейсе доступны:

- Dropbox
- Google Drive
- OneDrive
- Yandex Disk

## Что рекомендуется знать до подключения

Перед первым подключением лучше:

1. Создать и открыть локальное хранилище.
2. Добавить хотя бы одну тестовую или рабочую запись.
3. Сделать локальный бэкап.
4. Только после этого подключать cloud sync.

Так вы избежите ситуации, когда первая же удалённая операция окажется единственной копией данных.

## Как открыть настройки Cloud Sync

Cloud Sync настраивается для уже открытого хранилища.

Базовый путь:

1. Откройте нужное хранилище.
2. Перейдите в настройки текущего store.
3. Откройте страницу `Cloud Sync`.

На этой странице есть пошаговый сценарий подключения.

## Как подключить Cloud Sync к текущему хранилищу

Страница Cloud Sync разбита на три шага.

### Шаг 1. Выберите облачный сервис

Выберите провайдера, в котором будет храниться snapshot текущего хранилища.

### Шаг 2. Авторизуйте аккаунт

Если для выбранного сервиса ещё нет токена:

1. Нажмите `Авторизовать аккаунт`.
2. Завершите OAuth-авторизацию.

Если токен уже есть:

1. Выберите нужный аккаунт в поле `Подключаемый аккаунт`.
2. При необходимости используйте `Авторизовать ещё один аккаунт`.

### Шаг 3. Подключите хранилище

Нажмите `Подключить синхронизацию`.

После этого приложение:

- создаст облачную структуру для текущего хранилища
- свяжет текущий store с выбранным токеном
- подготовит первую snapshot-синхронизацию

## Что означают основные статусы

После подключения на странице Cloud Sync отображается состояние синхронизации и основное рекомендуемое действие.

### В облаке ещё нет snapshot

Смысл:

- удалённой версии для этого хранилища ещё нет

Обычно приложение предложит:

- `Загрузить локальную версию в облако`

Это нормальный первый сценарий после подключения.

### Локальная версия новее облачной

Смысл:

- вы изменили данные локально после последней синхронизации

Обычно приложение предложит:

- `Загрузить локальную версию в облако`

### Версии совпадают

Смысл:

- локальная и удалённая snapshot-версии согласованы

Доступное действие:

- `Проверить синхронизацию сейчас`

### Удалённая версия новее локальной

Смысл:

- в облаке уже есть более свежий snapshot, чем у вас локально

Обычно приложение предложит:

- `Скачать удалённую версию`

Перед этим разумно сделать локальный бэкап, если у вас есть важные несинхронизированные локальные изменения и вы не уверены в состоянии данных.

### Конфликт версий

Смысл:

- локальная и удалённая версии разошлись
- автоматический merge не используется
- нужно вручную выбрать основную сторону

Варианты:

- `Оставить локальную версию`
- `Принять удалённую версию`

Практическое правило:

- если уверены, что актуальные изменения находятся на текущем устройстве, оставляйте локальную версию
- если актуальные изменения уже загружены с другого устройства, принимайте удалённую версию

## Ручные действия, которые доступны на странице Cloud Sync

На подключённом хранилище доступны основные кнопки:

- `Обновить статус`
- `Проверить snapshot сейчас`
- `Переподключить аккаунт`
- `Отключить`

Когда использовать:

- `Обновить статус` если нужно просто перечитать текущее состояние привязки и версий
- `Проверить snapshot сейчас` если хотите форсировать проверку облачной версии
- `Переподключить аккаунт` если токен устарел, сменился аккаунт или нужна новая авторизация
- `Отключить` если хотите отвязать текущее хранилище от cloud sync

## Что происходит при повторной авторизации

Если OAuth-токен устарел или пропал, приложение может запросить ручную повторную авторизацию.

Обычно это значит:

- облачная привязка store сохранена
- сам токен больше недействителен или отсутствует на устройстве
- нужно заново пройти авторизацию и при необходимости переподключить аккаунт

В интерфейсе для этого используется действие:

- `Переподключить аккаунт`

## Как загрузить snapshot из облака при открытии хранилища

Если локального store ещё нет на устройстве или вы хотите восстановить удалённую версию:

1. Откройте экран `Открыть хранилище`.
2. Используйте сценарий `Скачать snapshot из Cloud Sync`.
3. Выберите провайдера и аккаунт.
4. Выберите удалённый snapshot.
5. Подтвердите импорт.

После импорта приложение может предложить:

- `Привязать к cloud sync`

Если это именно тот store, который вы хотите дальше синхронизировать, обычно имеет смысл согласиться.

## Как безопаснее работать с несколькими устройствами

Рекомендуемый сценарий:

1. Перед окончанием работы синхронизируйте актуальное состояние.
2. Перед открытием того же store на другом устройстве обновите статус cloud sync.
3. Если видите, что удалённая версия новее, сначала скачайте её, а не продолжайте вносить локальные изменения поверх старого состояния.
4. При сомнениях сначала создайте локальный бэкап, а уже потом принимайте удалённую или локальную сторону конфликта.

## Как правильно завершать работу

Если для хранилища включена синхронизация, лучше завершать работу штатным закрытием store, а не резким закрытием приложения.

Используйте:

- кнопку `Закрыть базу данных` в titlebar
- или кнопку `Закрыть БД` в drawer на dashboard

Это важно, потому что приложение умеет выполнять сценарий синхронизации перед закрытием хранилища.
