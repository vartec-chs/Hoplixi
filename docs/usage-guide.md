# Usage Guide

Краткий сценарий повседневной работы с Hoplixi: создание первого хранилища, добавление первой сущности, создание бэкапа и корректное закрытие.

Отдельно про облачную snapshot-синхронизацию: [Cloud Sync Guide](cloud-sync-guide.md)

## Создание первого хранилища

1. Откройте главный экран приложения.
2. Нажмите `Создать`.
   Подпись у действия: `Новое хранилище`.
3. Пройдите мастер создания хранилища.
4. Укажите:
   - имя хранилища
   - путь хранения
   - мастер-пароль
   - алгоритм шифрования базы, если приложение предлагает выбор
5. Завершите мастер кнопкой `Создать`.

После успешного создания приложение откроет хранилище и переведёт вас в dashboard.

## Как добавить первую сущность

После открытия хранилища вы попадаете в dashboard. В Hoplixi сущность это запись одного из типов: пароль, заметка, OTP, документ, файл, карта и т.д.

Базовый сценарий:

1. В dashboard выберите нужный тип сущности в текущем разделе.
   Самый частый первый вариант: `Пароли`.
2. Нажмите `Добавить`.
   На desktop и mobile это действие открывает форму создания записи для текущего типа.
3. Заполните обязательные поля.
4. При необходимости добавьте:
   - категорию
   - теги
   - иконку
   - пользовательские поля
5. Сохраните запись.

Если вы только начинаете, имеет смысл первой записью создать обычный пароль:

1. Откройте раздел `Пароли`.
2. Нажмите `Добавить`.
3. Заполните название, логин, пароль и URL при необходимости.
4. Сохраните запись.

После этого запись появится в списке текущего раздела.

## Когда делать бэкап

Рекомендуется создать первый бэкап сразу после:

- создания нового хранилища
- добавления первых важных записей
- массового импорта или заметных изменений

В интерфейсе есть два быстрых способа создать бэкап уже открытого хранилища:

- в `titlebar` на desktop кнопкой с иконкой `backup`
- в `drawer` на dashboard кнопкой `Бэкап`

Бэкап создаётся для текущего открытого хранилища.

## Как правильно закрывать хранилище

Хранилище лучше закрывать штатной кнопкой приложения, а не просто закрытием окна или принудительным завершением процесса.

Используйте один из двух вариантов:

- в `titlebar` кнопку `Закрыть базу данных` с иконкой выхода
- в `drawer` на dashboard кнопку `Закрыть БД`

Почему это важно:

- приложение успевает корректно завершить работу с базой
- если включена логика синхронизации перед закрытием, она сможет завершиться штатно
- снижается риск оставить рабочую сессию в неконсистентном состоянии

Важно:

- не путайте `Закрыть базу данных` с системной кнопкой закрытия окна `X` в titlebar
- если хотите сначала сохранить резервную копию, нажмите `Бэкап`, дождитесь завершения, и только потом закрывайте хранилище

## Короткий рекомендуемый сценарий

1. Создать хранилище.
2. Сразу добавить первую сущность.
3. Создать бэкап.
4. После завершения работы закрывать хранилище через `Закрыть базу данных` или `Закрыть БД`.
