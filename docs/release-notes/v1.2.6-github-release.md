# Hoplixi v1.2.6

Дата релиза: 2026-05-08

Этот релиз сфокусирован на безопасности облачной синхронизации и расширении возможностей обмена данными.

## Что нового

### ☁️ Облачная синхронизация (Cloud Sync)

- **Механизм Cloud Lock:** Добавлена система блокировок для предотвращения конфликтов. При открытии синхронизируемого хранилища создается удаленный файл `.lock`. Если хранилище уже открыто на другом устройстве, приложение предупредит об этом.
- **Интерфейс контроля доступа:** Добавлен блокирующий UI, который сообщает об активной блокировке другим экземпляром приложения. Вы можете либо выйти, либо продолжить работу на свой страх и риск.
- **Авто-очистка stale-блокировок:** Реализована логика автоматического освобождения блокировок, возраст которых превышает 24 часа.
- **Обновление визуализации:** Провайдеры облаков получили качественные PNG-логотипы, а интерфейс карточек и чипов в разделе синхронизации обновлен для их отображения.

### 🔑 Менеджер паролей и Шеринг

- **Универсальный шеринг:** Добавлено окно выбора конкретных полей записи для отправки через системное меню «Поделиться». Чувствительные поля (пароли) по умолчанию не выбраны для повышения безопасности.
- **Буфер LocalSend:** Выбранные поля теперь можно сохранить в локальный текстовый буфер для последующей отправки в активную сессию LocalSend.
- **Улучшенный UI:** Окно шеринга переведено на адаптивный `WoltModalSheet`, что обеспечило удобный скроллинг и корректное отображение на всех типах устройств.
- **Навигация по типам записей:** Управление отображаемыми типами сущностей в навигации вынесено в отдельное модальное окно, доступное напрямую из Dashboard.

### 🛠 Улучшения интерфейса и UX

- **Smooth UI:** Кнопки действий в окне шеринга и других компонентах переведены на `SmoothButton` для соблюдения единого дизайн-кода.
- **Оптимизация навигации:** Основное окно настроек хранилища (`StoreSettingsModal`) упрощено за счет выноса управления типами записей.

## Для разработчиков

- Добавлен `LocalSendBufferProvider` для хранения разделяемых данных только в памяти.
- Логика владения облачной блокировкой перенесена в `CurrentStoreCloudLockNotifier`.
- Обновлена документация `APP_CAPABILITIES.md`, ставшая полноценной картой подсистем приложения.
- В `GEMINI.md` закреплено требование обязательного ведения `CHANGELOG.md`.
