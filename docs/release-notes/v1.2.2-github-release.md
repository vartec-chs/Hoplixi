# Hoplixi v1.2.2

Дата релиза: 2026-04-29

## Что нового

### Password Generator

- В генератор паролей добавлено редактирование набора символов для каждой
  категории прямо из UI.
- Пользовательские наборы символов теперь сохраняются в профилях и
  восстанавливаются при их выборе.
- В `PasswordGeneratorWidget` вынесены отдельные виджеты в папку
  `lib/features/password_generator/widgets` без использования `part`.

### Password Manager

- В dashboard app bar добавлен пункт `Бэкап сейчас`, который запускает ручное
  создание бэкапа для открытого хранилища.
- В dashboard добавлена отдельная настройка для порога, при котором используются
  анимированные `SliverAnimatedList` / `SliverAnimatedGrid`.
- Добавлен общий switch для полного отключения анимаций dashboard: при
  выключении список, сетка и diff-обновления переходят в статичный режим.
- `CloseStoreSyncContent` и связанный mobile flow стали устойчивее и понятнее:
  обновлены тексты, добавлена анимация, улучшен статус overlay и закрытие через
  after-close sync.
- Мобильный `FloatingNavBar` получил более выразительный glass-стиль и мягкое
  затемнение поверх контента.

### Cloud Sync

- `CloudSyncHttpClient` подключён к `dio_smart_retry`, чтобы retry автоматически
  срабатывал для подходящих сетевых и HTTP status ошибок.
- Поведение retry и refresh flow сохранено для 401-сценариев.

### UI and Settings

- Dashboard settings теперь доступны прямо из `SettingsScreen`.
- Экран истории переведён на `WoltModalSheet` для фильтров, чтобы modal UX был
  единообразным.

## Важно перед обновлением

- Если вы хотите отключить лишние анимации на слабых устройствах, используйте
  новый switch в dashboard settings.
- Для ручного бэкапа хранилище должно быть открыто.

## Для разработчиков

- Dashboard settings теперь собраны в общую секцию настроек, без отдельного
  routed screen.
- Password generator UI разбит на отдельные файлы виджетов без `part`-файлов.
- Добавлен новый dashboard menu action для ручного бэкапа.
- `lib/main_db` был переписан на 60%: крупные части storage, manager, backup,
  close-sync и migration flows переведены на новую структуру и отдельные
  сервисы/провайдеры.
- Старые unit-тесты, завязанные на прежнюю архитектуру, удалены и будут
  переписаны под новую структуру `lib/main_db`.
