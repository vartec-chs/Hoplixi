# Hoplixi v1.2.0

Дата релиза: 2026-04-18

## Что нового

### Icon Packs (новая file-backed feature)

- Добавлены пользовательские SVG-паки иконок без изменения текущей таблицы
  icons.
- Поддержан импорт паков из zip-архивов и обычных папок.
- Реализован новый Wolt-based picker: выбор пака -> выбор SVG-иконки.
- Добавлен экран управления паками, новый маршрут /icon-packs и переход с
  главного экрана.

### Массовые действия в dashboard

- Добавлен режим multi-select через long-press по карточке.
- Добавлены bulk-операции: удаление, архивация, избранное, закрепление,
  назначение категории и тегов.
- На маленьких экранах действия компактно сворачиваются в popup-меню toolbar.

### Улучшения db_core и миграций

- Добавлен каркас версионированных миграций MainStore и централизованный
  migration runner.
- Расширена совместимость store_manifest: проверка manifestVersion,
  lastMigrationVersion и appVersion.
- Для устаревших хранилищ открыт сценарий backup -> migrate -> open.
- Для хранилищ из более новых версий клиента добавлено явное блокирование
  открытия (безопасный fail-fast).

### Надежность и безопасность

- Исправлены SQL-шаблоны history-триггеров.
- Улучшена надежность cloud sync: повтор HTTP-запроса после refresh токена в
  auth/expired сценариях.
- Проверено отсутствие логирования чувствительных данных в qr_scanner и очищено
  лишнее логирование в otp_form_provider.

### UI/UX улучшения

- ComponentShowcase адаптирован под мобильные экраны (Drawer вместо
  NavigationRail на узких экранах).
- Добавлен IndexedStack для сохранения состояния showcase-экранов.
- Обновлен Button Showcase: добавлены состояния Default, Disabled и Loading.
- Исправлены disabled/loading состояния в SmoothButton.
- Улучшен UX CategoryManager (стабильный скролл формы).
- В IconSourcePickerButton кнопка очистки перенесена в правый верхний угол
  карточки.

## Важно перед обновлением

- При открытии части старых хранилищ может потребоваться миграция.
- Рекомендуемый сценарий: создать backup, затем выполнить миграцию и открыть
  хранилище.
- Сравнение appVersion обновлено: суффикс +buildNumber больше не влияет на
  решение о миграции.

## Для разработчиков

- MainStoreManager декомпозирован: логика совместимости и синхронизации
  манифеста вынесена в отдельные сервисы.
- MainStore упрощен: установка history-триггеров и индексов вынесена в отдельные
  модули.
