создать модуль в папке lib\features\cloud_sync авторизация рассчитанный для десктопа и мобильных устройств (кроссплатформенный)
основные провайдеры будут Dropbox, Yandex, Google, OneDrive и возможно будут добавлены ещё учти это в архитектуре.

модуль разбить на 3 подмодуля:

1) подмодуль "app_credentials" это модуль хранения записных пользователем данных приложений для авторизации по типу (name, clint_id, client_secret)
может быть так чтоб есть несколько разных app credentials для одного типа провайдера учесть это. Будут и встроенные app credentials данные по одному для каждого провайдера описанные в отдельном файле в виде листа структур (этот лист может быть пустой зависит от резных версий изданий приложения).
все данные app credentials будут хранится в hive_ce: ^2.19.3 доступ в которой к записи можно получить только по ключу так как остальные данные зашифрованы
для app_credentials созать общий конфиг где будут заданы иконки для каждого типа провайдера желательно создать общий enum с доступными провайдерами 
учти этот момент в авторизации и в ui при создании пользователем своих app_credentials для OneDrive показывать в ui схему "для остальных"
если пользователь захочет создать своё приложения для входя на экране создания app_credentials показывать нужные url для правильной настройки своего приложения для авторизации и список scope (они будут прописаны в общем конфиге)

ui app_credentials:
1) основной экран список всех app_credentials отдельно отделены встроенные данные app_credentials если они есть их нельзя редактирова и удалять остальные созданные пользователем можно.
2) экран создания редактирования app_credentials

2) подмодуль "auth_tokens" используя hive_ce: ^2.19.3 мой самописный менеджер для Box создать структуру в которой после авторизации будут хранится все токены любых провайдеров

подмодуль auth_tokens должен предоставлять публичные методы для получения токена или токенов 
стутктура токена обазательно должна включать в себя access, опциональный refresh токены (так как не все провайдеры дают refresh токен) ну и другие данные

ui auth_tokens
1) экран всех токенов + модалка для просмотра всех дынных токена и возможность удалить токен

3) подмодуль "auth"
для мобильных устройств реализовывать будем с помощью этих библиотек: 
	1) flutter_appauth: ^12.0.0 (все остальные)
	2) google_sign_in: ^7.2.0 (Google)
для десктопа реализовать отдельный loopback flow авторизацию используя хост 127.0.0.1 
требования к loopback flow (OAuth 2.0 для десктопных приложений) возможность отменить запущенную авторизацию и проброс явных ошибок для пользователя

важно на мобильных устройствах требуются разные схемы переадресации например: 
для Dropbox
<data android:scheme="db-ulc6d2kyzhp8di2" android:host="oauth2redirect" />
для OneDrive
<data android:scheme="msauth" android:host="com.example.hoplixi_test_auth" android:path="/dXthLzqPWWG4DvCSzuBcRbPb7WA%3D" />
для остальных 
<data android:scheme="hoplixiauth" android:host="oauth2redirect" />


ui auth
1) модалка для авторизции со списком всех провайдеров при выборе нужного запускается процесс авторизации и перебрасывает на экран процесса авторизации и запоминать прошлый путь лочиться на этом экране и при успешной авторизации или отмене возвращаться на прошлый экран
использовать lib\routing\router_refresh_provider.dart и lib\routing\router.dart
2) экран самого процесса авторизации спинер и кнопка отмены 
создать провайдер для записи про статус авторизации и данные какой провайдер был выбран для авторизации Dropbox или другие и id токена для записнного в hive_ce если у спешный результат 



общие требования:
папки у каждого подмодуля с их именем и в них папки:
models файлы с freezed модулями с json сериализацией
services файлы сервисов реализация бизнеслогики
providers riverpod провайдеры обертка над сервисами и работа в ui
screens экраны этого подмодуля
widgets виджеты экранов
utils вспомогательные функции (опционально)

lib\core\services\hive_box_manager.dart - мой самописный менеджер для работы с hive_ce: ^2.19.3 который позволяет работать с разными боксами и имеет встроенные функции для шифрования данных
hive_box_manager_quickstart.dart - пример использования hive_box_manager для быстрого старта
hive_box_manager_examples.dart - примеры использования hive_box_manager для разных кейсов
рабочий пример используемый в проекте - lib\main_store\services\db_history_services.dart

документацию про flutter_appauth: ^12.0.0 и google_sign_in: ^7.2.0 лежит в этой директории с названием "flutter_appauth_docs.md" и "google_sign_in_docs.md" если что-то нужно узнать ищи с помощью mcp context7
